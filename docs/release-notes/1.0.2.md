# ECodex 1.0.2 发布说明

ECodex 1.0.2 聚焦“常驻后台工作台”和“AI 开发复盘证据”。这个版本让终端在关闭窗口后更适合长期运行，强化命令完成 / 失败提醒，并补齐 Session Vault 的失败 loop 预览能力。

## 推荐下载

| 产物 | 适用场景 | 说明 |
|---|---|---|
| `ecodex-win-x64-sc` | 大多数 Windows 10/11 用户 | self-contained app 目录，不需要共享 .NET runtime。 |
| `ecodex-cli-win-x64` | 自动化脚本和 CI | 仅命令行包，提供 `ecodex` 命令。 |
| Inno Setup 安装器 | 传统桌面安装 | 简体中文安装 / 卸载界面，创建快捷方式，卸载保留 `%USERPROFILE%\.ecodex`。 |
| MSIX 包 | 企业分发 | 适合受管环境。 |

详细步骤见 [安装](../installation.md)。

## 新特性

- **托盘常驻**：关闭按钮或最小化会隐藏到系统托盘，后台终端和通知继续工作；托盘菜单可选择保留或终止后台终端。
- **低噪声通知**：PowerShell hook 能携带 workspace / surface / pane 上下文，后台命令完成、失败或 Codex 等待输入时进入未读中心，并支持 Toast 点击跳转。
- **默认 skills 种子**：App 启动时从安装目录 `default-skills` 安全复制到 `%USERPROFILE%\.agents\skills`，同名目录跳过，不覆盖用户已有 skill。
- **多行输入**：所有 ECodex Terminal 默认支持 `Ctrl+Enter` 输入换行，普通 `Enter` 仍提交命令。
- **失败 loop 预览**：Session Vault 可以从 terminal transcript 生成可复制的失败命令、transcript 和诊断证据预览。
- **AgentConversation recorder seam**：Core 层新增 runtime recorder seam，为后续把 Agent 消息写入可复盘证据包打基础；当前不接真实 LLM 或 UI。

## 运维与维护者改进

- 新增 `.ecodex/ecodex.example.json` dogfood 示例，覆盖本仓 build、unit test、docs build、status 和 health 命令。
- 新增 `scripts/smoke-ecodex-v2.ps1`，用于手动验证 status、workspace、pane 与 browser 最小闭环。
- 新增 `scripts/smoke-toast-activation.ps1`，用于记录 Toast 展示、点击恢复、pane 聚焦和 Codex 等待输入提醒证据。
- 新增 `scripts/release-evidence.ps1`，用于汇总发布前 build、test、docs、perf、doctor 和 release workflow 证据路径。
- PR 模板新增可选 Handoff 区块，便于 Agent 中断或交接时记录验证、风险、下一步和回滚点。

## 升级注意事项

- 替换文件前建议停止 `ecodex-app.exe`、`ecodex-daemon.exe` 和旧终端会话，避免 Windows 文件锁。
- 安装后运行 `ecodex doctor` 和 `ecodex setup status` 检查 PATH、daemon、WebView2、PowerShell hook 与配置目录。
- 卸载与更新默认保留 `%USERPROFILE%\.ecodex`，不会删除现有 session、resume、command log 或用户 skills。
- Session Vault 的 Agent 消息区在真实 Agent runtime 接线前可能为空；当前失败 loop 预览仍以命令日志、transcript 和诊断证据为主。

## 发布门槛

- 发布前仍按 P0 = 0、P1 <= 3 且 P1 必须有 workaround 的门槛验收。
- 如果安装、Toast、WebView2 或 ConPTY 行为异常，优先参考 [故障排查](../troubleshooting.md) 和 [发布就绪](../release-readiness.md)。
