# ECode 1.0.0 Release Notes

ECode 1.0.0 is the first stable Windows-native SuperTerminal release. It brings
terminal workspaces, browser panes, scriptable automation, session restore, and
installer-ready distribution into one desktop workflow.

## Recommended Download

Choose the artifact that matches how you want to install ECode:

| Artifact | Best for | Notes |
|---|---|---|
| `ecode-win-x64-sc` | Most Windows 10/11 users | Self-contained app folder; no shared .NET runtime required. |
| `ecode-cli-win-x64` | Automation and scripting hosts | CLI-only package for `ecode` commands. |
| Velopack setup/feed | Users who want app updates | Supports `ecode update check` and `ecode update install`. |
| Inno Setup installer | Traditional desktop installs | Adds shortcuts and keeps `%USERPROFILE%\.ecode` during uninstall. |
| MSIX package | Enterprise deployment | Optional app package route for managed environments. |

See [Installation](../installation.md) for detailed install, update, and
uninstall instructions.

## What Is New

- **Terminal workspaces**: create multiple workspaces, surfaces, and split panes
  for high-density terminal workflows.
- **Session persistence**: keep layout, terminal snapshots, browser metadata, and
  trusted resume bindings in `%USERPROFILE%\.ecode`.
- **Browser surfaces**: open WebView2 browser panes next to terminals, navigate
  with toolbar controls, and persist browser URLs/history.
- **Scriptable Browser API**: automate snapshot, locator queries, click, fill,
  hover, press, eval, screenshots, cookies, storage, console, dialog, download,
  and highlight flows through `ecode browser` and `ecode.v2`.
- **Short refs and v2 CLI**: address windows, workspaces, surfaces, panes, and
  notifications with `window:N`, `workspace:N`, `surface:N`, and `pane:N`.
- **Windows integration**: configure PATH, PowerShell completion, Windows
  Terminal profiles, setup diagnostics, update checks, and release artifacts.

## First Run Checklist

After installing:

```powershell
ecode doctor
ecode setup status
ecode completion powershell
```

Open the app, create a workspace, and try:

```powershell
ecode workspace create --name demo
ecode pane split right
ecode browser new https://example.com
ecode status --json
```

## Upgrade Notes

- Stop running `ecode-app.exe`, `ecode-daemon.exe`, and old terminal sessions
  before replacing files.
- Back up `%USERPROFILE%\.ecode\session.json` and
  `%USERPROFILE%\.ecode\resume.json` if you rely on custom resume bindings.
- Existing runtime data in `%USERPROFILE%\.ecode` is retained by uninstall and
  update flows.
- Run `ecode doctor` after installation to verify ConPTY, WebView2, PATH,
  daemon, and config directory status.

## Known Limits

- Windows 10 1809 / build 17763 or newer is required for ConPTY.
- WebView2 Runtime is required for browser surfaces.
- Release builds publish an `ecode-perf-report` artifact with cold-start,
  `ecode status`, `browser.snapshot`, and session-save budget rows.
- Browser automation intentionally returns `not_supported` for non-P0 network,
  viewport, geolocation, trace, and screencast operations listed in
  [Browser API](../browser-api.md).
- Release blockers are tracked in [Release Readiness](../release-readiness.md);
  the 1.0 gate requires P0 = 0 and P1 <= 3 with documented workarounds.
